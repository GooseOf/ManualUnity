Объекты

В Unity есть готовые примитивные объекты, которые можно добавить в свой проект.

На каждом объекте есть свои компоненты

Компоненты можно удалять и добавлять по желанию и необходимости

В Unity есть множество стандартных компонентов, отвечающих за самые разные вещи (звук, отрисовка, передвижение, физика и т.д.)

На каждом объекте обязательно есть компонент Transform. Он отвечает за позицию и направление объекта в пространстве, а так же его размеры

Компоненты MeshFilter и MeshRenderer. Включение и отключение компонентов

MeshFilter отвечает за форму объекта, MeshRenderer за его текстуру, цвет и отрисовку

Компоненты можно включать и отключать

Как видно, при отключении компонента отвечающего за отрисовку - объект перестал быть видимым (зеленые линии от компонента Collider и в игре их не видно)

Так же и с другими, например при отключении коллайдера - объект перестанет сталкиваться с другими объектами

Физика

Компонент RigidBody заставляет объект подчиняться законам физики (Слева объект с компонентом RigidBody, справа объект без него)

В окне инспектора видно силы действующие на объект, его скорость и прочее

Компонент коллайдер добавляет объекту физическое тело (То есть без коллайдера объект подчиняющийся физике не будет врезаться в другие объекты).
Коллайдеры можно подобрать по форме, настроить по полигонам (для 2Д), взять форму меша из MeshFilter и т.д.

Далее наглядно показана работа коллайдера (слева объект с коллайдером, справа без)

У компонента отвечающего за свет есть несколько типов свечения

Spot Light имитирующий фонарь

Point Light свет из точки во все стороны

Так же свету можно менять цвет

Системы частиц

Система частиц используется для имитирования огня, дыма, тумана и прочих спецэффектов. В системе частиц можно настроить внешний вид частиц, их размер, скорость, время жизни, столкновения с объектами и т.д.

Примеры использования систем частиц

Выхлоп машины

Сияние атмосферы

Реактивная струя шаттла

Жидкость

Жидкость со столкновениями

Собственные компоненты

Для начала нужно создать свой скрипт

В скриптах можно включать и выключать, двигать, создавать, уничтожать объекты и компоненты. Влиять на их значения, например меняя цвет или массу объекта и т.д.

Здесь есть два метода Start, который вызывается единожды при появлении объекта, а так же метод Update, который вызывается каждый кадр

Так же есть методы которые вызываются когда объект стал видимым для игрока, когда объект столкнулся с другим объектом, когда игрок кликнул по объекту, когда объект был уничтожен и т.д.

Все эти методы пришли от класса MonoBehaviour по умолчанию унаследонного нашим скриптом

Немного поработав со скриптом можно получить такой результат

Однажды написав скрипт его можно добавлять на множество объектов, выше скрипт был только на одном из кубов, ниже скрипт был добавлен на оба

Из скрипта так же можно получать другие компоненты и влиять на них, управлять ими, включать или отключать, добавлять новые или удалять имеющиеся

Вот пример готового скрипта, реализующего управление машиной с аркадной физикой

В инспекторе можно задавать публичные поля класса не меняя скрипт (Максимальная скорость, скорость вращения и ускорение)

Немного поигравшись на сцене можно получить такой результат
